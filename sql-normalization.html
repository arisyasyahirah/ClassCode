Understanding Normalization

Normalization is the process of organizing data in a database to reduce redundancy (duplication) and improve data integrity.

The Problem: Redundancy

Imagine a table where we store student info and their course info in one place. If a student takes 3 courses, their name and address are repeated 3 times. This is "unnormalized."

The Normal Forms

1st Normal Form (1NF)

Each table cell must contain a single value (atomic), and each record must be unique.

2nd Normal Form (2NF)

Must be in 1NF, and all non-key columns must depend on the entire primary key. This usually involves splitting tables.

3rd Normal Form (3NF)

Must be in 2NF, and there should be no transitive dependencies (columns shouldn't depend on other non-key columns).
