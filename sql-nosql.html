Beyond Relational Databases

NoSQL (Not Only SQL) databases provide a mechanism for storage and retrieval of data that is modeled in means other than the tabular relations used in relational databases (SQL).

[Image of SQL vs NoSQL comparison showing tabular data structure vs JSON document data structure]

Core Differences

  • Schema: SQL uses fixed schemas; NoSQL uses dynamic schemas for unstructured data.
  • Scaling: SQL is vertically scalable (bigger server); NoSQL is horizontally scalable (more servers).
  • Structure: SQL is table-based; NoSQL can be document-based, key-value pairs, or graph-based.

Document-Based Storage (JSON)

The most popular form of NoSQL uses JSON (JavaScript Object Notation) to store data. This allows for nested data and flexible fields.
