Mastering Reusable Logic

Functions prevent "Spaghetti Code." Instead of writing the same 10 lines of code every time you need to calculate a tax or validate a form, you wrap them in a function.

Parameters vs. Arguments

Parameters are the variables listed in the function definition. Arguments are the real values you pass when you call it.

function multiply(a, b) { // a, b are parameters
  return a * b;
}
multiply(5, 10); // 5, 10 are arguments

The Return Value

Think of a function as a vending machine. You put in a coin (argument), it does work, and it returns a snack (value).

Note: Once a function hits return, it stops immediately.

Detailed Breakdown: Why Use Functions?

  1. DRY Principle: "Don't Repeat Yourself." If you need to change logic, you only change it in one place.
  2. Organization: You can give names to complex tasks (e.g., processPayment()) making code readable.
  3. Scoping: Functions create a protective bubble. Variables created inside a function (Local Scope) don't interfere with variables outside.

Anonymous & Arrow Functions

Modern JS uses shorter syntax for functions that are only used in one place:

// Standard
function(x) { return x * 2; }

// Arrow Function (Modern)
const double = (x) => x * 2;
