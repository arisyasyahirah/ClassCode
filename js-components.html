Dynamic Website Components & Architecture

A dynamic website is an interactive system where content changes based on user input or logic. This requires a specific environment to run.

Detailed Note: The Client-Server Ecosystem

  1. The Request: The user enters a URL in the Browser (Client). The browser sends an HTTP request to the server.
  2. The Server: A computer hosting the site. It checks if the request needs "Parsing" (processing code like PHP).
  3. The Response: The server sends back Web Objects (HTML, CSS, JS).
  4. The Parser: The browser's engine parses the code and executes the JavaScript Behavior.

The Three Functional Layers

Modern web development separates code to make it maintainable:

  • Structure (HTML): The "Bones."
  • Presentation (CSS): The "Paint and Decor."
  • Behavior (JavaScript): The "Electricity and Plumbing."

Pro-Tip: The "Interpreted" Advantage

JavaScript is an interpreted language. This allows us to use the Document Object Model (DOM) to change the page instantly!
