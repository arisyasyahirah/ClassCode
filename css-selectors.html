Mastering Element Targeting

CSS selectors allow you to pick exactly which HTML elements you want to style. However, when multiple rules try to change the same element, Specificity acts as the referee to decide who wins.

Advanced Selector Types

Beyond simple IDs and Classes, you can combine selectors to be more precise:

  • Descendant Selector (space): Targets elements inside another element (e.g., div p targets all paragraphs inside a div).
  • Child Selector (>): Targets only direct children.
  • Pseudo-classes (:): Targets elements in a specific state, like :hover (when the mouse is over it) or :nth-child(2).

The Specificity Point System

Browsers calculate a "score" for every CSS rule. The rule with the highest score is applied to the element.

Selector Type Points Example
Inline Style 1000 style="color: red;"
ID Selector 100 #header
Class, Attribute, Pseudo-class 10 .menu, :hover
Element (Tag) Selector 1 h1, p, div

Student Study Tips

  • The "0-0-0-0" Rule: Write your specificity as four numbers (Inline-ID-Class-Element). A 0-1-0-0 (ID) always beats a 0-0-2-0 (two classes).
  • Avoid IDs for Styling: Professional developers prefer classes. Using IDs makes your CSS too "strong," making it hard to override later.
  • Last Resort: Only use !important if you are fighting with a third-party library. It is considered bad practice in clean code.
  • Read Right-to-Left: CSS selectors are read by the browser from right to left. ul li a means "Find all links, that are in list items, that are in a unordered list."
