Understanding Stored Procedures

A Stored Procedure is a prepared SQL code that you can save, so the code can be reused over and over again. Think of it as a function for your database.

Benefits of Procedures

  • Performance: The code is compiled once and stored, making execution faster.
  • Security: Users can execute the procedure without needing direct access to the underlying tables.
  • Consistency: Ensures that complex logic is applied the same way every time it is called.

Creating and Executing

To create a procedure, we use CREATE PROCEDURE. To run a procedure that has already been saved, we use the EXEC command.

Example Syntax

CREATE PROCEDURE SelectAllStudents AS SELECT * FROM Students GO;

To call this procedure later:

EXEC SelectAllStudents;

Question: Can a Stored Procedure accept input values to filter results?

Answer: Yes! These are called Parameters. You can pass values into a procedure to make the SQL logic dynamic (e.g., EXEC GetStudentByID @ID = 1;).
