HTML Links & Images: Connecting and Visualizing Content

Links and images are fundamental elements of web pages. Links create connections between pages and resources, while images add visual appeal and information.

Anchor Element (<a>)

Basic Link Syntax

The <a> (anchor) element creates hyperlinks to other web pages, files, email addresses, or locations on the same page.

Example:

Visit our homepage for more information.

<a href="https://www.example.com" target="_blank">Visit Example.com</a>

Link Attributes

Attribute Values Description
href URL Specifies the link destination
target _blank, _self, _parent, _top Where to open the linked document
rel nofollow, noopener, noreferrer Relationship between documents
download filename Downloads the file when clicked
title text Extra information about the link

Types of Links

<!-- External link -->
<a href="https://www.google.com" target="_blank" rel="noopener">Google</a>

<!-- Internal link -->
<a href="about.html">About Us</a>

<!-- Email link -->
<a href="mailto:contact@example.com">Email Us</a>

<!-- Phone link -->
<a href="tel:+60123456789">Call Us</a>

<!-- Download link -->
<a href="document.pdf" download="my-document.pdf">Download PDF</a>

<!-- Anchor link -->
<a href="#section2">Jump to Section 2</a>

Image Element (<img>)

Basic Image Syntax

The <img> element embeds images in web pages. It's a self-closing tag.

Example:

<img src="image.jpg" alt="Description of image" width="300" height="200">

Image Attributes

Attribute Values Description
src URL Source file path
alt text Alternative text (required for accessibility)
width/height pixels Image dimensions
title text Tooltip text
loading lazy/eager Lazy loading behavior
srcset URL + width descriptor Responsive images for different screens

Figure and Figcaption

Use <figure> and <figcaption> for semantically marked up images with captions.

<figure>
        <img src="landscape.jpg" alt="Beautiful mountain landscape">
        <figcaption>Fig. 1 - A beautiful mountain landscape at sunrise</figcaption>
    </figure>

Image Formats and Optimization

Common Image Formats

Format Best For File Extension Compression
JPEG Photographs, complex images .jpg, .jpeg Lossy
PNG Graphics, logos, transparency .png Lossless
GIF Simple animations, small graphics .gif Lossless
SVG Vector graphics, icons .svg Vector (scalable)
WebP Modern web (better compression) .webp Both lossy/lossless

Responsive Images

Using srcset and sizes

Provide different image files for different screen sizes:

<img src="image-small.jpg"
         srcset="image-small.jpg 300w,
                 image-medium.jpg 600w,
                 image-large.jpg 1200w"
         sizes="(max-width: 600px) 300px,
                (max-width: 1200px) 600px,
                1200px"
         alt="Responsive image example">

Picture Element

Use <picture> for art direction and format support:

<picture>
        <source media="(min-width: 768px)" srcset="large.jpg">
        <source media="(min-width: 480px)" srcset="medium.jpg">
        <img src="small.jpg" alt="Responsive art direction">
    </picture>

Image Links and Image Maps

Image as a Link

<a href="gallery.html">
        <img src="thumbnail.jpg" alt="View gallery">
    </a>

Image Maps

Create clickable areas on an image:

<img src="workplace.jpg" alt="Workplace" usemap="#workmap">

    <map name="workmap">
        <area shape="rect" coords="34,44,270,350" alt="Computer" href="computer.html">
        <area shape="rect" coords="290,172,333,250" alt="Phone" href="phone.html">
        <area shape="circle" coords="337,300,44" alt="Coffee" href="coffee.html">
    </map>

Best Practices

  • Always include descriptive alt text for images
  • Use appropriate image formats and compress images
  • Specify width and height attributes to prevent layout shifts
  • Use rel="noopener" for external links opening in new tabs
  • Make link text descriptive (avoid "click here")
  • Implement responsive images for different screen sizes
  • Use lazy loading for images below the fold
  • Keep file names descriptive and SEO-friendly
  • Optimize image delivery with CDNs
  • Consider using modern formats like WebP
