HTML5 Forms: Interactive User Input

HTML forms are essential for collecting user input. HTML5 introduced new input types, attributes, and elements that make forms more powerful and user-friendly.

Form Element (<form>)

Basic Form Structure

The <form> element creates a container for form controls that can be submitted to a server.

Example:



<form action="/submit-form" method="POST">
        <!-- Form controls go here -->
    </form>

Form Attributes

Attribute Values Description
action URL Where to send form data
method GET, POST HTTP method for submission
target _blank, _self Where to display response
autocomplete on, off Enable/disable autocomplete
novalidate Boolean Disable validation

Input Element (<input>)

Common Input Types

Type Description Example

HTML5 Input Attributes

Attribute Description Example
placeholder Hint text inside input
required Makes field mandatory
autofocus Auto-focus on page load
pattern Regex validation pattern
min/max Minimum/maximum values
step Increment step for numbers
multiple Allow multiple values
list Associates with datalist

Other Form Elements

Textarea (<textarea>)

For multi-line text input:

<textarea rows="4" cols="50" placeholder="Enter text...">
    Default text here
    </textarea>

Select & Option (<select> & <option>)

For dropdown selection:

<select name="country">
        <option value="">Select country</option>
        <option value="MY">Malaysia</option>
        <option value="SG">Singapore</option>
        <option value="ID">Indonesia</option>
    </select>

Datalist (<datalist>)

Provides autocomplete suggestions:

<input list="browsers" name="browser" placeholder="Choose browser">
    <datalist id="browsers">
        <option value="Chrome">
        <option value="Firefox">
        <option value="Safari">
        <option value="Edge">
    </datalist>

Fieldset & Legend

Groups related form elements:

<fieldset>
        <legend>Personal Information</legend>
        <!-- Form fields here -->
    </fieldset>

Label Element (<label>)

Associates text with form controls for accessibility:

<!-- Method 1: Using for attribute -->
    <label for="username">Username:</label>
    <input type="text" id="username" name="username">

    <!-- Method 2: Wrapping the input -->
    <label>
        <input type="checkbox" name="subscribe">
        Subscribe to newsletter
    </label>

Buttons

Type Description Example
<button type="submit"> Submits the form
<button type="reset"> Resets form fields
<button type="button"> General purpose button
<input type="submit"> Submit button (old style)

Form Validation

Client-Side Validation

HTML5 provides built-in validation:

<input type="email" required placeholder="Valid email required">
    <input type="text" pattern="[A-Za-z]{3,}" title="At least 3 letters">
    <input type="number" min="18" max="100">
    <input type="url" required>

Validation Attributes

  • required: Field must be filled
  • minlength/maxlength: Text length limits
  • min/max: Numeric value limits
  • pattern: Regular expression pattern
  • title: Tooltip for validation message

Best Practices

  • Always use <label> elements for accessibility
  • Use appropriate input types (email, tel, etc.)
  • Provide helpful placeholder text
  • Use required attribute for mandatory fields
  • Group related fields with <fieldset>
  • Use client-side validation but validate on server too
  • Make forms keyboard accessible (tab order)
  • Provide clear error messages
  • Consider mobile users with appropriate input types
  • Test forms thoroughly across browsers

Solution:
<div>
  <label for="userEmail">Email Address:</label>

Challenge 2: Create a password field with minimum length requirement.

Solution:
<div>
  <label for="userPassword">Password:</label>
  <input type="password" id="userPassword" name="password"
      placeholder="At least 8 characters"
      required minlength="8">
</div>

Challenge 3: Create a form with a range slider that displays its value.

Solution:
<div>
  <label for="volume">Volume: <span id="volumeValue">50</span></label>

</div>
